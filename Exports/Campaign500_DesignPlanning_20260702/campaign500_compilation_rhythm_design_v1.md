# Campaign500 编排节奏设计 V1

生成日期：2026-07-02

## 目标

这份文档先定义 500 关需要什么样的体验，再决定怎么生成和替换。昨天那版最大问题不是候选质量，而是把“链条数/候选等级”当成主轴，导致每个 slot 的职责不够清楚。

新的编排原则：

> 500 关不是难度排序表，而是“10 关微循环 + 50 关章节弧 + 500 关长期变化”的体验系统。

玩家持续玩的动力来自五件事：

1. 有爽感：普通关能顺畅推进，能让玩家觉得自己变强。
2. 有变化：风格、链条语言、画布比例、形状/洞洞节奏都要轮换。
3. 有读题：困难关不能只靠更多箭头，要要求玩家换区域、换方向、识别依赖。
4. 有峰值：特别困难/极难关要成为阶段记忆点。
5. 有释放：峰值之后要有恢复关，避免连续压迫。

## 三层节奏

### 1. 10 关微循环

每 10 关是一个小节奏单位。它不要求每组都完全同构，但应该有类似的体验职责：

- 开场：恢复、进入状态、或者轻量变化。
- 中段：普通关逐步抬高，并插入一种风格/链条语言变化。
- 后段：困难 slot 做读题测试。
- 末尾：特别困难/极难/shape/hole 形成记忆点，随后下一组回落。

10 关内允许小回退。比如第 4 段普通均值约 50 时，可以有一关回到 40 多，用来制造呼吸，而不是所有关线性上涨。

### 2. 50 关章节弧

每 50 关应该有一个主学习目标，例如：

- 1-50：建立规则、安全感和基础变化。
- 51-100：形成稳定循环，玩家理解普通/困难/峰值的节奏。
- 101-200：开始扩大风格差异，降低纯 flow 爽关占比。
- 201-300：前 300 的质量核心段，更多读题、依赖和区域切换。
- 301-500：高级变化和精选回顾，允许更高难度但仍要安排释放。

### 3. 500 关长期变化

长期不是单纯“越后越大”，而是逐步增加：

- solve-order 深度。
- 跨区域干扰。
- 链条语言辨识度。
- 局部连消被打断的频率。
- 画布、密度、形状、洞洞带来的空间阅读变化。

## Difficulty Code 的职责

原模板比例保留：普通/恢复 314，困难 94，特别困难 59，极难 33。

### Code 1：普通/恢复

普通不是单一类型，至少分四种职责：

- RecoveryFlow：峰值后的顺滑恢复，爽感优先。
- NormalPractice：练习当前段的主语言，难度稳定。
- LanguageVariation：换链条语言或画布，但不显著加压。
- BridgeRamp：普通中的上坡位，为后面的困难做铺垫。

普通关不能长期都像 flow，也不能长期都像 peel。前期 flow 占比可以高，后期逐渐下降，但每个高压段后仍需要恢复。

### Code 2：困难

困难的职责是让玩家“读一下”，不是单纯多链条。

困难 slot 应该至少满足一个：

- 区域切换明显。
- 方向切换明显。
- 局部连续消除被打断。
- 有轻量依赖或 choke。
- 画布/密度相对本 10 关普通有明确抬高。

### Code 3：特别困难

特别困难是当前 10/20 关内的峰值。它应该比困难有明显差别：

- 更深依赖。
- 更强的跨区读题。
- 更大的有效负载。
- 更明确的风格记忆点，例如 peel 分层、longchain 主干、hub 汇聚。

### Code 4：极难

极难是章节或半章节记忆点。它不应该频繁出现，也不应该只是“特别困难 + 更多链条”。

极难应该有明确主题：

- ChapterPeak：章节 boss。
- ExtremeRead：高读题密度。
- StyleCulmination：某个风格语言的阶段总结。
- VisualPeak：shape/hole 与难度共同形成记忆点。

## Shape / Hole 的定位

shape 和 hole 是 category，不是简单难度码。它们可以承担峰值，但不是拿来填 normal peak slot 的工具。

- ShapeAnchor：视觉变化和记忆点，适合做阶段亮点。
- HoleSpatialAnchor：空间约束变化，适合做节奏换气或空间读题。

shape/hole 如果出现在困难或峰值附近，要把它当成这个 10 关段的 peak 之一，normal slot 不必再强行补一个同等峰值。

## 风格和链条语言

风格是 solve-order 拓扑，链条语言是单条/局部路径的形态。

### 风格

- Flow：连续推进，适合爽关和恢复。
- Peel：层剥释放，适合稳定中难度。
- LongChain：长依赖主干，适合读题峰值，但链条数要加权看。
- Hub：汇聚/分支控制，适合困难和特别困难。
- Maze：约束场，当前先作为后期研究，不作为稳定量产主力。
- Mixed：不限风格和链条语言，只受质量门槛约束，作为自然变化池。

### 链条语言

- Curve：弯曲蛇形，空间覆盖自然，但容易显得同质。
- Rail：直链/折线为主，阅读更清楚，也能打破全蛇形观感。
- Spine：主干长链，适合 LongChain 和 peak。
- Patch：短链密集局部块，适合制造读题和局部干扰。
- Mixed：不限定语言，用质量门槛筛自然组合。

每个风格应支持多种链条语言；每个 10 关段要避免连续 3 关同一风格/语言。

## 难度不只看链条数

后续排布使用有效负载，而不是 raw chain count。

建议概念公式：

```text
effectiveLoad =
  rawChainCount
+ canvasBandBonus
+ longChainBackboneBonus
+ readDemandBonus
+ shapeHoleContextBonus
- recoveryFlowDiscount
```

LongChain 的 raw chain count 往往偏低，但如果画布大、主干长、选择少、依赖深，它的 effectiveLoad 应该高于表面链条数。

## 质量门槛

所有正式候选必须先过硬门槛：

- solved = true。
- 能被 GreedyValidator 在预算内清空。
- 有初始可消链。
- 难度分不为 0。
- 不重复占格、不越界、不非法链。

再按 slot 职责筛：

- coverage：符合当前段画布/密度目标。
- choices：普通可以顺，困难不能过度爆炸，极难也不能不可读。
- local run：同一区域连续消除不能长期占优。
- direction risk：同方向扫穿要限制。
- region switch：困难以上要有明确跨区。
- axis drift：方向变化要有节奏，不是纯单轴推土机。
- collapse：不能过早坍缩成单区清扫。
- repetition：同一 10 关内不能连续重复风格、语言、画布手感。

## 生产流程

后续不要直接拿候选池排序塞 slot。改成：

1. 从模板生成 500 个 slot spec。
2. 每个 slot spec 写清楚职责、难度码、category、目标有效负载、风格偏好、链条语言偏好、硬性禁忌。
3. 按 slot spec 生产或挑选候选。
4. 候选进入池时记录来源、风格、链条语言、raw chain、effectiveLoad、quality gates。
5. 排布时先保证前 300 关质量，后 200 可保留旧关或逐步替换。
6. 每 20/50 关进 Unity 体感审查一次，不等到 500 全塞完再看。

## 当前结论

下一步不是继续补 80 关，而是先生成 `Campaign500 Slot Spec V1`：

- 保留模板的 order/category/difficultyCode/shape/hole 信息。
- 为每个 slot 增加 `slotDuty`、`targetExperience`、`styleTarget`、`chainLanguageTarget`、`effectiveLoadBand`、`replacementPriority`。
- 用它指导后续生产和导入。
